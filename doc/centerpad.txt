*centerpad.txt* Center your single lonely buffer easily!

Centerpad centers the current editing window by placing blank, unfocusable
padding buffers on the left and right.

USAGE                                                               *centerpad*

Run `:Centerpad` or `:lua require("centerpad").toggle()` to toggle centering.
`:Centerpad` accepts one or two numbers:

- `:Centerpad 20`       sets both pads to 20 columns
- `:Centerpad 10 20`    sets left pad to 10 and right pad to 20

EDGE CASES                                                   *centerpad-edge*

- Ignored buffers (`terminal`, `quickfix`, `help`, `nofile`, `centerpad`) skip
  enablement.
- Floating windows never become the centered main window.
- Invalid widths below `1`, above `500`, empty, or non-numeric notify an error
  and leave existing pads, config, and state unchanged.

CONFIGURATION

Default padding: 25 columns on each side. Override via `setup`:

```lua
require("centerpad").setup({
  leftpad = 25,
  rightpad = 25,
  enable_by_default = false,
})
```

Example keybindings in Lua:

```lua
-- using the command
vim.api.nvim_set_keymap(
  "n",
  "<leader>z",
  "<cmd>Centerpad<cr>",
  { silent = true, noremap = true }
)

-- using the lua function
vim.api.nvim_set_keymap(
  "n",
  "<leader>z",
  "<cmd>lua require('centerpad').toggle()<cr>",
  { silent = true, noremap = true }
)
```

Example keybindings in Vimscript:

```vim
" using the command
nnoremap <silent><leader>z <cmd>Centerpad<cr>

" using the lua function
nnoremap <silent><leader>z <cmd>lua require('centerpad').toggle()<cr>
```

STATE GLOBALS

- `vim.g.centerpad_enabled` serves as the primary enabled-state flag.
- `vim.g.center_buf_enabled` remains mirrored for one release as a
  deprecated bridge.

MIGRATION

Replace `vim.g.center_buf_enabled` with `vim.g.centerpad_enabled`. The legacy
global still syncs for this release, then support drops.

SCOPE

- Blank side padding around the current editing window.
- Focus prevention on pads, with focus redirected to the main window.
- State validation and automatic recovery.

NON-GOALS

- Zen-mode or distraction-free layout management.
- Feature parity with other centering or zen plugins.
